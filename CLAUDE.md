# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

This is a YouTube video research and scriptwriting workspace serving **two channels** (see below). Each video project lives in its own folder under its channel's `videos/` directory (e.g., `channels/Amreet Talks/videos/China takes over AI/`). The research and scripting methodology is documented in `docs/research-framework.md`.

## Channels

The repo is organized by channel under `channels/` — see [`channels/README.md`](channels/README.md). Keep the two channels separate (don't cross-post); it keeps the audience signal clean for the algorithm.

- **Amreet Talks** (`channels/Amreet Talks/`) — AI/tech business & strategy explainers. Faceless, long-form, Remotion motion-graphics. See [`channels/Amreet Talks/channel.md`](channels/Amreet%20Talks/channel.md).
- **Amreet Aint** (`channels/Amreet Aint/`) — gaming, **Hindi/Hinglish**, ~855 subs, PS5+PC, facecam-in-corner; cinematic story + **horror**. The growth bet is a weekly Hindi-horror series — see [`channels/Amreet Aint/growth-strategy.md`](channels/Amreet%20Aint/growth-strategy.md) (lane, exclusions/dupes, the make-today pick, packaging, and the "Fresh Hindi Horror, Har Hafte" workflow).

## Available MCP Tools

The project `.mcp.json` configures these servers (all launched via `npx`):

- **google-trends** — compare keywords, interest by region, related queries
- **playwright** — browser automation for scraping articles and verifying sources
- **remotion** — drive the `remotion-all/` motion-graphics project (scoped to `./remotion-all`)

> For cited factual research, prefer tools that show their sources (e.g. Perplexity, or the
> WebSearch/WebFetch tools) over raw LLM output. YouTube transcripts are pulled via the
> `youtube-transcript-analysis` skill (yt-dlp), not an MCP server. Finding a channel's own
> recent videos/streams (rather than analyzing a known URL) is the `youtube-channel-discovery`
> skill — also `yt-dlp`, no API key or MCP server needed.

### Blender (video editing) — Windows Claude only

A `blender` MCP (`uvx blender-mcp`) drives Blender's Video Sequence Editor to cut clips, layer
memes, add zoom punch-ins, and render. It is **not** in this `.mcp.json`: Blender and the MCP run on
**Windows**, so it's registered in *Windows* Claude user scope (`claude mcp add blender -s user --
uvx blender-mcp`) and reaches Blender on `localhost`. Editing runs from Windows Claude opening this
repo via the `\\wsl.localhost` share. See [`docs/blender-video-editing.md`](docs/blender-video-editing.md)
for the VSE recipes and the `/mnt/f` ⇔ `F:\` path rule.

## Research & Scripting Workflow

Follow `docs/research-framework.md` for the full process. The short version:

1. **5W1H (Kipling Method)** — generate research questions before searching anything: What, Who, When, Where, Why, How. Not all six matter equally for every topic.
2. **Research** — Google, Wikipedia, YouTube, books. Use Tavily/Perplexity for cited answers. Use Google Scholar / Consensus for scientific topics. Avoid Quora/Reddit as primary sources.
3. **Go deep** — horizontal logic (multiple angles at the same level) + vertical logic (drilling to root causes on the most important questions).
4. **Script structure**:
   - Hook: shocking stat, story, or teaser clip — goal is curiosity
   - Main content: basics first (What/When/Who), then surprising/deep angles
   - Only rabbit-hole on questions *central* to the video's core topic
   - Link points with transitions that raise the next question — not abrupt cuts

## Project Folder Convention

Each video idea gets its own directory under its channel — `channels/<Channel>/videos/<Video>/`.

**Amreet Talks (AI/explainer)** — place inside:
- `idea.md` — seed idea or premise
- `research.md` — sourced research notes with links
- `script.md` — final script draft

**Amreet Aint (gaming/horror)** — lighter, edit-first projects:
- `idea.md` — thesis, hook, and packaging (titles / thumbnail / ~10-min beats / SEO tags)
- `cut-list.md` — cold-open + beat skeleton, filled from the recording's watchback pass
- The first cut is generated by `scripts/firstcut_fcpxml.py` (FCPXML → DaVinci Resolve, free). Media
  (VODs/clips/renders) lives on the F: drive (`/mnt/f`), not the repo.

## Motion Graphics

Animated graphics for the videos are built in the **`remotion-all/`** Remotion project — one
folder per video under `remotion-all/src/<VideoName>/` (e.g. `src/China/`), with one composition
clip per script section. Follow `docs/motion-graphics-guide.md` for the structure, the reusable
component library, the Remotion gotchas, and the visual/craft design language. The generic
Remotion API knowledge lives in the `remotion-best-practices` skill.
